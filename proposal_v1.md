# 收敛版研究设想 v1

## 一句话版本

本研究拟构建一个由 LLM 负责信息理解、regime 识别、模型编排与解释生成的自适应预测 Agent，用于国际原油价格的实时预测与极端事件压力测试。

## 推荐摘要写法

本研究拟提出一个面向国际原油价格预测的 LLM 驱动自适应 Agent 框架。与将大语言模型直接作为预测器的做法不同，本文将 LLM 定位为多源异质信息环境下的编排器与解释器：在每个预测时点，Agent 联合结构化基本面指标、GDELT 地缘政治事件流以及从 IEA 月报、OPEC 声明和重要政策讲话中提取的高级语义特征，识别当前市场所处的主导 regime，并据此自适应调用因果诊断工具、调整特征配置和选择预测模型。框架最终输出的不仅是价格预测值，还包括结构化的驱动因素归因、不确定性来源识别以及模型失效条件预警。样本期暂定为 2020-01-01 至 2026-03-13，覆盖负油价、战争冲击、供应链扰动以及 2025-2026 年中东冲突升级等多类极端情境，可用于检验该框架在 regime 迁移和尾部事件下的稳健性。本文的预期贡献在于：提出一种由 LLM 实现的预测编排机制；构建一个融合结构化、事件型与文本型信号的 regime-aware 油价预测框架；并通过可审计解释报告提升复杂预测系统的透明度与可用性。

## 推荐研究问题

1. 在多次结构突变和极端事件背景下，LLM 编排的 regime-aware 预测框架能否优于固定模型与固定特征集？
2. 在引入文本语义特征和事件流后，Agent 是否能够更早识别驱动机制切换？
3. 因果诊断模块究竟提高的是预测精度、解释稳定性，还是模型在尾部事件下的鲁棒性？

## 框架拆解

### 1. 环境感知模块

- 输入：库存、产量、期限结构、美元指数、VIX、GDELT 事件统计、文本抽取语义特征。
- 输出：当期 regime 概率，至少区分为基本面驱动、地缘驱动、预期/投机驱动。

### 2. 因果诊断模块

- 目标不是证明结构性真因果，而是给出当期局部可用的因果线索。
- 可选方法：滚动 Granger、PCMCI、transfer entropy、局部 DAG 搜索、事件窗口因果诊断。
- 输出：当前窗口内候选驱动变量集合及其稳定性评分。

### 3. 自适应预测模块

- 根据 regime 和因果诊断结果，动态选择模型或动态融合模型。
- 可比较的预测器：线性模型、树模型、TFT、PatchTST、时间序列基础模型、MoE。
- 输出：点预测、区间预测、方向预测概率。

### 4. 解释与预警模块

- 不建议输出原始 chain-of-thought。
- 建议输出结构化 explanation card：
- 主导驱动因素
- 关键证据来源
- 不确定性来源
- 模型失效条件
- 与上一期决策相比的变化

## 最关键的方法约束

1. 所有特征必须按可获得时间对齐，禁止回看未来文本和修订后数据。
2. regime 判定必须只基于 t 时点可观测信息。
3. 因果诊断只能作为动态筛选和解释工具，不能夸大为已识别真实结构因果。
4. 所有评估必须采用 rolling-origin 或 walk-forward 设计。

## 建议的实验设计

### 主实验

- 目标：比较 Agent 编排框架与固定模型框架的 out-of-sample 表现。
- 指标：RMSE、MAE、directional accuracy、tail-event hit rate、区间覆盖率。

### 消融实验

1. 去掉文本语义特征。
2. 去掉 GDELT 事件流。
3. 去掉因果诊断模块。
4. 去掉 regime switching，仅使用统一模型。
5. 将 LLM 编排替换为人工规则编排。

### 压力测试

1. 2020-04 负油价阶段。
2. 俄乌冲突升级阶段。
3. 2025-2026 中东冲突升级阶段。

## 审稿人最可能质疑的问题

1. LLM 到底提供了什么增量价值，而不是把已有流程文字化？
2. regime 是真实可识别状态，还是事后解释标签？
3. 文本抽取得到的语义变量如何验证可靠性？
4. 解释模块给出的归因是否稳定，还是每次都变？
5. 极端事件样本很少，结果是否只是 case study？
